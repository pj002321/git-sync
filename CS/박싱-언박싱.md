값 타입과 참조 타입 간 변환 메커니즘
값 타입-> 참조 타입 : 박싱
참조 -> 값 : 언박싱

왜 쓰는지?
- 값 타입을 object기반 API, 컬렉션, 인터페이스에서 다루기 위해 사용.

언제 쓰는지?
- object, interface 타입 파라미터를 사용할 때
- 제네릭을 쓰지 못하는 오래된 API 사용 시
- 로그, 고용 컨테이너, 범용 처리 로직에서 값 타입을 다룰 때

장점
- 값 타입을 참조 타입처럼 범용적으로 처리 가능하고 기존 object 기반 API와 호환 가능

단점
- 힙 할당 발생, GC비용 증가
- 언박싱 시 형 변환 오류 가능

예시
```csharp

int value = 10;
object boxed = value; // Boxing

int unboxed = (int)boxed; // Unboxing

```

메모리
```
Stack (value type)
   ↓ Boxing
Heap (object)
   ↓ Unboxing
Stack (value type)

```
