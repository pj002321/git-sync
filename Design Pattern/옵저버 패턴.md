## 옵저버 패턴 (Observer Pattern)

객체의 상태가 변경되면, 그 객체에 등록된 다른 객체들에게 자동으로 알림을 보내는 행동 패턴.

왜 쓰는가?
- 객체의 상태 변화가 발생했을 때 이를 의존 객체들에게 자동으로 전파하면서 결합도를 낮추기 위해 사용한다.

장점
- 객체 간 결합도가 낮고, 새로운 반응 객체를 기존 코드 수정 없이 쉽게 확장할 수 있다.

단점
- 알림 흐름이 명시적이지 않을 때가 있고, 옵저버가 많아지면 성능 문제가 생김

구조
- 상태를 관리하는 Subject와 이를 구독 해제하며 알림 받는 Observer로 구성
- Action, Event
